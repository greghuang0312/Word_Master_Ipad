## 2. 回滚步骤（Rollback）

当上线后出现关键故障（登录不可用、数据串读、复习结果错误）时：

1. 立即停止新版本分发（下架当前测试包或冻结更新入口）。
2. 将客户端切回上一个稳定包（若无稳定包，回退到最小可用壳层版本）。
3. 暂停写操作路径（添加、复习提交），保留只读浏览能力。
4. 核查 Supabase：RLS 策略、用户隔离、关键表写入权限。
5. 修复后先在 iPad 测试账号做冒烟通过，再恢复发布。

## 3. 冒烟测试建议（待人工执行）

4. 库页删除词条后即时生效。
5. 我的页 API Key 保存与退出登录可用。

## 4. 发布结论
