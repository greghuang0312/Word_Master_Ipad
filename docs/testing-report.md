# Word Master 测试验证报告（Phase 4）

- 日期：2026-03-11
- 关联实现：`docs/development-implementation.md`
- 测试阶段目标：功能验收、端到端流程验证、风险确认

## 1. 验证范围

1. 登录流程（账号密码登录、退出）
2. 复习流程（会/不会路径与阶段变化）
3. 添加流程（DeepSeek 候选、手写兜底、重复合并）
4. 库管理（展示、删除、阶段显示）
5. 统计视图（总量/进行中/逾期/已掌握、分布、时间轴）
6. 我的页面（API Key 输入保存、退出）
7. 横竖屏切换稳定性

## 2. 自动测试执行情况

1. 当前环境为 Windows，缺少 `xcodebuild`，无法执行 iPad 自动化测试命令。
2. 自动测试状态：`PENDING_MANUAL`
3. 需在 iPad/Xcode 环境补跑以下命令：
   - `xcodebuild test -scheme WordMaster -destination "platform=iOS Simulator,name=iPad (10th generation)"`

## 3. 人工测试建议清单（iPad 侧）

1. 使用管理员预建账号登录，确认可进入 5 Tab。
2. 在“添加”输入中文，测试候选点选与手写英文保存。
3. 用重复英文再次保存，确认触发合并更新提示。
4. 在“复习”测试：
   - 直接点卡片 => 阶段 +1
   - 点击“英文翻译”后点卡片 => 回退阶段 1，次日复习
5. 在“库”删除一个词条，确认立即消失。
6. 在“统计”核对四个指标与阶段分布。
7. 在“我的”保存 API Key、退出登录。
8. 在竖屏/横屏切换中检查页面状态是否保持。

## 4. 当前结论

1. 代码实现已完成，待 iPad 侧人工验收闭环。
2. 由于自动测试缺失，当前测试结论为：`PENDING_MANUAL`。

## 5. 风险与回滚

1. 风险：未在真实 iPad 运行，可能存在 UI/生命周期问题。  
回滚：按模块逐步回退最近改动，优先保留核心复习算法与数据模型。
