# Word Master iPad 需求文档（Phase 1）

- 项目代号：Word Master
- 文档日期：2026-03-11
- 目标平台：iPadOS（Swift Playgrounds）
- 网络策略：仅联网使用，不支持离线
- 屏幕方向：支持竖屏与横屏（支持旋转切换）

## 1. 问题定义

高中生背单词存在“学了就忘、复习不连续、进度不可视”的问题。  
本项目通过艾宾浩斯阶段复习机制，提供“今日待复习 -> 结果判定 -> 阶段推进/回退 -> 可视化统计”的闭环，降低遗忘率并提升复习执行率。

## 2. 目标用户与核心场景

### 2.1 用户画像

- 主要用户：高中生
- 使用环境：iPad（课堂/课后）

### 2.2 核心场景

- 学生每日打开“复习”页，按系统给出的当日队列完成背诵。
- 学生在“添加”页输入中文，获取英文候选后选定或手写，形成新词条。
- 学生在“统计”页查看学习进度与阶段分布，理解复习规则。

## 3. MVP 边界

### 3.1 In Scope（V1）

- iPadOS 单端
- 支持竖屏与横屏两种方向
- 账号密码登录（仅登录，不注册）
- Supabase 作为后端数据存储与鉴权
- DeepSeek 作为“中文 -> 英文候选”服务
- 五个页面：复习、添加、库、统计、我的

### 3.2 Out of Scope（V1 不做）

- iPhone / Mac 适配
- 离线模式
- 用户自助注册
- 多义词条（一个中文对应多个英文）

## 4. 功能需求与验收标准

## FR-01 登录

- 描述：首次进入需账号密码登录；账号由管理员预先在 Supabase 配置。
- 验收标准：
1. 未登录时无法访问核心页面。
2. 输入正确账号密码可登录成功。
3. V1 无注册入口。

## FR-02 复习页

- 描述：以“中文释义卡片”驱动复习。
- 规则：
1. 当日复习队列 = 所有 `next_review_date <= 今天` 的词条（包含逾期）。
2. 用户直接点击卡片（未点“英文翻译”）视为“会”，进入下一阶段。
3. 用户点击“英文翻译”后再点卡片，视为“不会”，立刻回退到阶段 1。
4. “不会”回退后，下一次复习日期设为第二天。
- 验收标准：
1. 卡片点击后可切换到下一词条。
2. “会/不会”路径的阶段变化与日期变化符合规则。
3. 队列会包含历史逾期词，并在清空前持续显示。

## FR-03 添加页

- 描述：输入中文后调用 DeepSeek 返回英文候选，用户选择其一；若不满意可手写英文。
- 规则：
1. 单义模型：一个中文词条最终对应一个英文单词。
2. 重复英文词（同一账号）触发“合并更新”，并提示用户。
- 验收标准：
1. 输入中文可获得候选英文列表。
2. 可点选候选保存，也可手写英文保存。
3. 遇到重复英文词不新增重复记录，而是执行合并更新并提示。

## FR-04 库页

- 描述：展示用户全部词条，含中英文、阶段信息，并支持删除。
- 验收标准：
1. 列表显示英文单词 + 中文释义 + 当前阶段。
2. 每条记录可删除且结果即时反映在列表中。

## FR-05 统计页

- 描述：提供学习阶段报告、阶段分布、时间轴对照与规则说明。
- 统计项：
1. 总单词量
2. 进行中
3. 逾期
4. 已掌握
- 验收标准：
1. 可展示阶段分布图（基于 1/2/4/7/15/30 天阶段）。
2. 可展示艾宾浩斯复习时间轴对照表。
3. 明确展示“自动任务顺延”与“忘记后回退阶段 1”规则说明。

## FR-06 我的页

- 描述：支持 DeepSeek API Key 输入/更新，支持退出登录。
- 验收标准：
1. 可输入并保存 API Key（具体存储实现在架构阶段确定）。
2. 可退出当前账号并回到登录态。

## FR-07 iPad 交互与方向适配

- 描述：所有页面需同时支持 iPad 竖屏与横屏，并严格遵循 Apple iPad 交互规范。
- 验收标准：
1. 五个页面在竖屏与横屏下均可正常展示与操作，无关键控件遮挡或错位。
2. 设备旋转时页面布局可平滑切换，当前流程状态不丢失（如复习中的当前卡片）。
3. 导航、触控反馈、层级结构与常见控件行为遵循 Apple Human Interface Guidelines（iPad）。

## 5. 艾宾浩斯规则（已确认）

- 阶段数：6 阶段
- 阶段间隔天数：`1/2/4/7/15/30`
- 新增词条首次复习日期：第二天
- 复习“会”：阶段 +1（上限阶段 6）
- 复习“不会”：回退到阶段 1，下一次复习日期为第二天

## 6. 数据与隔离要求

- 后端：Supabase
- 隔离原则：不同用户账号数据完全隔离（RLS）
- 关键数据对象：
1. 词条（用户、中文、英文、阶段、下次复习日期）
2. 复习记录（复习时间、结果、阶段变更）
3. 用户设置（如 API Key 配置）

## 7. 非功能与设计约束

- 交互与视觉：严格遵循 Apple Human Interface Guidelines（iPad 端）
- 方向适配：竖屏与横屏均为正式支持形态，不允许仅单方向可用
- 稳定性：关键流程（登录、拉取今日队列、保存复习结果）失败时须有明确提示
- 安全性：账号鉴权与数据访问必须受 Supabase Auth + RLS 约束

## 8. 风险与回滚预案（Phase 1）

## 8.1 主要风险

1. DeepSeek 返回不稳定导致候选质量波动
2. 阶段推进/回退规则在实现中出现口径偏差
3. RLS 配置错误导致用户间数据串读

## 8.2 回滚预案

1. 若 AI 候选异常，保留“手写英文”兜底路径，保障可用性
2. 若复习算法异常，先回退到“固定阶段更新”稳定版本
3. 若鉴权或隔离异常，先下线写操作，仅保留只读并排查策略
